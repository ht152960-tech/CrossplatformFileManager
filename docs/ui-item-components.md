# UI Item Components

## 当前阶段执行原则

- 本文档是长期 UI 条目规范，不代表每次修改都要一次性实现全部组件。
- V3 以已定稿的 Medium 首页作为首页条目与信息层级主标准。
- 先抽 Design Tokens，再抽基础视觉容器，再抽标签和按钮。
- 业务组件 `FileItem` / `RecommendedFileItem` / `RecentFileItem` / `FileTypeItem`
  可以在更多页面稳定后再抽取。
- 不得为了抽 token 或复用组件破坏 Compact / Medium / Expanded 已验证效果。
- 如果发现现有页面与本文档不一致，先记录 TODO，不要顺手大改。

- `W`：宽屏 / `expanded`
- `M`：中屏 / `medium`
- `C`：窄屏 / `compact`

规则：

- Medium 为 V3 主参考布局。
- Expanded 为 Medium 的扩展版。
- Compact 为 Medium 的压缩版。
- 所有新页面优先完成 Medium 设计，再派生 Expanded 与 Compact。

- 中屏保留核心信息，减少辅助信息。
- 窄屏优先保留识别和操作必需信息。

## 通用文件条目 `FileItem`

适用：

- 全部文件列表
- 搜索结果列表
- 最近添加条目的基础形态
- 推荐条目的基础形态
- 文件详情页中的相关文件列表

必须显示：

- 文件图标 / 缩略图
- 文件名.扩展名
- 添加时间、文件大小、标签属于常规信息。
- 不同变体可以根据空间和页面语义选择显示常规信息。

可选显示：

- 文件类型
- 最近打开时间
- 文件路径
- 更多操作按钮
- 标签数量
- 文件状态提示

不应显示：

- 假数据
- 未实现功能的按钮
- 与当前页面无关的复杂统计
- 无法解释的推荐分数

点击行为：

- 点击文件图标 / 缩略图：直接打开文件。
- 点击条目其他区域：进入文件详情页。
- 该规则适用于 `W`、`M`、`C`，不同页面不应随意改变。

缩略图遮罩规则：

- 视频文件：播放键遮罩。
- 音频文件：播放或音频提示遮罩。
- 图片文件：图片类型提示或轻量遮罩。
- 普通文档：使用文件类型图标。
- 遮罩应轻量，不破坏缩略图识别。

响应式规则：

- `W`：显示完整信息，可补充路径、最近打开时间或操作按钮。
- `M`：保留文件图标 / 缩略图、文件名.扩展名、添加时间、文件大小、标签；空间不足时收敛标签数量或标签数量提示。
- `C`：优先保留文件图标 / 缩略图、文件名.扩展名、添加时间；文件大小、标签可以隐藏或收纳。

## 最近添加条目 `RecentFileItem`

本质：

- `RecentFileItem` 是 `FileItem` 的紧凑变体，不是全新类型。

适用：

- 首页最近添加卡片
- 其他需要展示最近添加文件的小型列表区域

必须显示：

- 文件图标 / 缩略图
- 文件名.扩展名
- 添加时间

- `RecentFileItem` 作为紧凑变体，不应被强制显示完整 `FileItem` 的全部常规信息。

可选显示：

- 文件大小
- 文件类型

不应显示：

- 文件路径
- 推荐理由
- 复杂操作按钮
- 过多标签
- 与最近添加无关的统计信息

视觉要求：

- 每一行外层应填满父容器宽度。
- 所有条目的视觉长度应一致。
- 右侧时间区域应有稳定宽度。
- 文件名过长时应截断，不能撑破布局。
- 背景和边框应轻，不要像厚重输入框。

点击行为：

- 点击文件图标 / 缩略图：直接打开文件。
- 点击条目其他区域：进入文件详情页。

## 推荐条目 `RecommendedFileItem`

本质：

- `RecommendedFileItem` 是 `FileItem` 的增强变体。
- V3 以 Medium 首页推荐条目为主标准。
- 推荐身份通过紧贴文件名的轻量推荐徽标表达，不使用推荐理由文字。

适用：

- 首页智能推荐主面板
- 推荐页主列表
- 未来与推荐算法相关的文件列表

必须显示：

- 文件图标 / 缩略图
- 文件名.扩展名
- 紧贴文件名的轻量推荐徽标
- 标签摘要；有标签时使用 `#标签名` 文本，不使用类型 Chip 或标签 Chip

可选显示：

- 文件大小
- 最近打开时间
- 添加时间
- 更多操作按钮

不应显示：

- 文件类型 Chip
- 推荐理由文字
- 推荐条目标签 Chip
- 无法解释的神秘分数
- 假推荐理由
- 未实现的 AI 分析说明
- 过于复杂的统计内容

推荐算法可以继续使用时间间隔规律、打开后的后继关系等真实信号，但条目
不把内部评分或推断原因直接渲染为说明文字。

点击行为：

- 点击文件图标 / 缩略图：直接打开文件。
- 点击条目其他区域：进入文件详情页。
- 不新增独立“打开”按钮。

视觉原则：

- 可以比普通文件条目更突出，但不能脱离文件条目的视觉家族。
- 推荐徽标只作轻提示，不与文件名争夺视觉焦点。
- 标签使用单行 `#文本` 摘要，超出空间时截断或使用剩余数量提示。

## 标签条目 `TagItem` / `TagChip`

`TagItem` 与 `TagChip` 分开约束。

统一显示与操作语义：

- 所有正式标签在 UI 中统一显示为 `#标签名`。
- 适用范围包括搜索条件中的正式标签、标签页、详情页文件标签、首页常用标签、推荐条目标签摘要、普通文件条目标签摘要、详情页添加标签候选。
- 临时搜索关键词不显示 `#`，例如 `接口`、`合同`、`resume`。
- `×` 只表示彻底删除对象本身。
- `−` 表示从当前上下文移除，不删除对象本身。
- `×` 和 `−` 都统一放在 chip 右上角，不放在 chip 正右侧。
- Android Compact 下 chip 和右上角操作按钮要略微增大，提升可点击性。
- 点击热区可以大于视觉圆；视觉圆不能过大遮挡文字。
- 纯展示摘要只展示 `#标签名`，不显示删除或移除按钮。

### `TagItem`

适用：

- 标签列表
- 首页标签入口
- 标签管理页

必须显示：

- 标签字符串，显示为 `#标签名`
- 有多少个文件拥有这个标签

可选显示：

- 删除按钮；仅彻底删除正式标签时使用右上角 `×`
- 编辑入口
- 颜色标记
- 标签状态

不应显示：

- 搜索关键词伪装成正式标签
- 未保存的临时条件
- 与标签无关的统计

点击行为：

- 点击正式标签：进入标签筛选或标签详情。
- 删除按钮只在可编辑场景出现，并且只用于彻底删除标签对象。
- 编辑入口只在明确支持编辑的页面出现。

视觉原则：

- 正式标签表示长期存在的数据，视觉上应更稳定、更肯定。
- 正式标签可使用较实的背景或较稳定的紫色层级。

### `TagChip`

适用：

- 文件条目
- 文件详情
- 标签摘要

必须显示：

- 正式标签名，显示为 `#标签名`

默认不显示：

- 文件数量

可选显示：

- 右上角操作按钮
- 编辑入口
- 颜色标记
- 标签状态

不应显示：

- 搜索关键词伪装成正式标签
- 未保存的临时条件
- 与标签无关的统计

点击行为：

- 点击正式标签：进入标签筛选或标签详情。
- 详情页从当前文件移除标签时使用右上角 `−`，不删除标签对象本身。
- 标签管理页彻底删除标签时使用右上角 `×`。
- 编辑入口只在明确支持编辑的页面出现。

视觉原则：

- 正式标签表示长期存在的数据，视觉上应更稳定、更肯定。
- 正式标签可使用较实的背景或较稳定的紫色层级。

## 搜索条件 / 临时搜索标签 `SearchTokenChip`

本质：

- `SearchTokenChip` 表示用户在搜索过程中形成的当前搜索条件。
- 当前搜索条件可能来自正式标签，也可能来自临时搜索关键词。
- 临时搜索关键词不是正式标签，不会写入文件标签数据。

适用：

- 搜索框下方
- 搜索条件区域
- 用户输入关键词后生成的临时搜索标签列表
- 标签页点击、首页搜索输入、搜索页输入后形成的当前搜索条件列表

必须显示：

- 正式标签条件显示为 `#标签名`
- 临时搜索关键词显示原始关键词，不带 `#`

建议显示：

- 右上角 `−` 移除按钮
- “关键词”或搜索图标提示

不应显示：

- 文件数量，除非搜索结果已明确计算
- 正式标签的文件数量
- 让用户误以为它已被保存为标签的样式

点击行为：

- 点击 `−`：移除该搜索条件。
- 点击主体：可以聚焦或高亮该条件，但不进入标签详情。
- 不得把临时搜索关键词写入文件标签数据。

规则：

- 搜索条件必须统一带 `−` 移除按钮。
- `−` 统一放在标签右上角。
- 点击 `−` 时，不应触发标签主体点击行为。
- `−` 视觉上要清楚，但不能过重。
- `−` 样式应与当前项目中采用的右上角操作样式保持一致。
- 不得在不同页面中一会儿放右侧、一会儿放右上角。
- 当前搜索条件必须始终显示 `−` 移除按钮。
- 无论来源是标签页点击、首页搜索输入，还是搜索页输入，都使用统一的当前搜索条件 chip 布局。
- 正式标签条件带 `#`，临时关键词不带 `#`。

与正式标签的关系：

- 同属一个视觉家族，但必须有区别。
- 正式标签：更实、更稳定，代表长期保存的数据。
- 临时搜索关键词：更轻、更透明，带 `−`，代表当前搜索条件。

## 效果图使用原则

- 效果图只作为视觉风格参考。
- 不得复制其中假数据、固定统计值、固定文件名或未实现功能。
- 颜色、圆角、间距应沉淀为 token，而不是在页面中硬编码。

## 文件条目中的标签显示规则

- 最多直接显示 3 个标签。
- 超过 3 个时，只显示前 3 个。
- 剩余未显示的标签必须用数量提示说明，例如 `+2`、`还有 2 个`、`+2 个标签`。
- V3 推荐条目使用 `#标签名` 文本摘要，不使用 `TagChip`。
- 其他普通文件条目和标签摘要仍可按页面语义使用 `TagChip`。
- 不得因为标签过多撑破条目布局。
- 不得在宽屏、中屏、窄屏中使用完全不同的标签数量规则。
- 窄屏空间不足时，可以只显示 1-2 个标签，但仍必须用剩余数量提示说明未显示数量。
- 标签排序优先使用当前已有逻辑；如果没有明确排序逻辑，保持原始顺序。

## 文件类型条目 `FileTypeItem`

适用：

- 首页文件类型卡片
- 文件页类型筛选
- 类型统计区域

必须显示：

- 类型图标
- 类型名称
- 有多少个文件属于这个类型

可选显示：

- 占比条
- 占比百分比
- 类型说明

不应显示：

- 没有真实来源的统计
- 复杂图表，除非页面明确需要
- 与类型筛选无关的操作

点击行为：

- 进入该类型的筛选结果，或在当前页面应用类型筛选。
- 具体行为应根据页面已有逻辑决定，但不能在不同页面中表现得互相矛盾。

视觉原则：

- 文件类型条目应比文件条目更轻。

文件扩展名统一映射：

- 音频：`mp3`、`wav`、`flac`、`aac`、`ogg`、`m4a`
- 视频：`mp4`、`mkv`、`avi`、`mov`、`wmv`、`webm`
- 图片：`png`、`jpg`、`jpeg`、`gif`、`webp`、`bmp`、`svg`
- 文档：`pdf`、`doc`、`docx`、`ppt`、`pptx`、`xls`、`xlsx`、`txt`、`md`
- 不属于以上映射的扩展名统一归入“其他”。

## 查看全部入口

适用：

- 最近添加卡片
- 标签卡片
- 文件类型卡片

必须显示：

- 位于卡片标题右侧的明确入口文案，例如 `查看全部`

可选显示：

- 轻量箭头图标
- 轻量 hover 状态

不应显示：

- 卡片底部固定 Footer
- 额外统计
- 未实现功能按钮
- 过亮的独立底色条
- 与卡片主体割裂的亮色框

点击行为：

- 跳转到对应完整页面，或执行当前已有的查看全部逻辑。

视觉原则：

- 查看全部是标题区的次级文字入口，不是独立主按钮。
- 同类卡片的入口位置和视觉层级必须统一。

## 上传入口 `UploadEntry`

语义：

- `UploadEntry` 是上传文件的主入口，W / M / C 共享业务语义。
- 不同尺寸可以使用适合当前布局的视觉变体，但不能改变上传行为。

变体：

- Compact 首页允许使用 FAB 作为主要上传入口。
- Medium 首页使用弱按钮上传入口，保持低存在感，不挤占首页主内容。
- Expanded 首页可使用侧栏上传入口。
- 三者保持业务一致，仅视觉权重不同。

约束：

- 不得为了统一外观强行让 Compact FAB 与 Medium 弱按钮使用相同布局。
- 不得新增重复上传入口。

## 颜色使用原则

1. 页面背景以深蓝黑和黑紫为主。
2. 卡片背景保持暗紫黑，不要偏蓝。
3. 主操作按钮使用紫色。
4. 选中态使用紫色系，不使用高饱和科技蓝。
5. 统计和类型图表可使用蓝紫、粉紫、橙色辅助，但不能喧宾夺主。
6. 正文使用白色。
7. 说明文字使用灰白。
8. 边框和高光应克制，避免霓虹感过强。
9. footer 不应使用过亮背景。
10. 列表项背景不应比卡片主体更抢眼。
